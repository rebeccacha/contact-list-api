REST Resources

Resource descriptions: {domain}/ContactList/rest/{path}
E.g. Retrieve all contacts: GET http://localhost:8080/ContactList/rest/contact/

Project Progress

Unit testing not complete.
Resource paths tested with the Postman Google Chrome extension.
There were sometimes problems in Postman if the Content-Type HTTP header value was present instead of absent.

The "Create" action uses the POST HTTP method and can be tested with an HTML form with a file-upload field. The "Update" action uses the PUT method and cannot be created through HTML alone.

Data is assumed to be correctly formatted.
Parameters should be able to be excluded to create null values in the database.
The PUT (update) method replaces the entire contact record, not just single fields.
Poorly formatted input is not well-checked and may cause the application to crash.

Searches are currently case-sensitive.
